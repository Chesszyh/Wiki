    Purpose and Scope¶

    This document provides an overview of the Linux kernel's storage and I/O subsystems, covering the path from user-space I/O operations down to physical storage devices. Three major components are documented:

    1. io_uring - Modern asynchronous I/O interface providing low-latency, high-throughput I/O operations
    2. Btrfs - Advanced copy-on-write filesystem with features like snapshots, checksumming, and software RAID
    3. UFS - Universal Flash Storage host controller driver for mobile and embedded storage devices

    For detailed information about each subsystem:

    • io_uring implementation details: see io_uring Asynchronous I/O
    • Btrfs filesystem internals: see Btrfs Copy-on-Write Filesystem
    • UFS hardware controller details: see UFS Storage Controller

    This page focuses on how these subsystems integrate within the broader Linux storage stack and their architectural relationships.

    Zero-Copy Networking¶

    io_uring includes a zero-copy receive path (ZCRX) that bypasses traditional kernel buffering for high-performance networking:

    • Interface Queue Management: struct io_zcrx_ifq manages per-queue state io_uring/zcrx.c479-493
    • Memory Areas: struct io_zcrx_area represents DMA-capable memory regions io_uring/zcrx.c409-478
    • Network Integration: Direct integration with page_pool for zero-copy packet reception io_uring/zcrx.c35-56

    Sources: io_uring/zcrx.c1-850

    Write Path in Btrfs¶

    The Btrfs write path implements copy-on-write semantics with several optimization strategies:

    1. Inline Extents: Small files stored directly in B-tree metadata fs/btrfs/inode.c458-566
    2. Delalloc: Delayed allocation for better contiguous allocation fs/btrfs/inode.c401-427
    3. Compression: Transparent compression during write fs/btrfs/inode.c521-538
    4. COW Processing: run_delalloc_cow() handles actual copy-on-write fs/btrfs/inode.c123-126

    Tree Logging for Fast fsync¶

    Btrfs provides fast fsync through a tree logging mechanism that avoids full transaction commits:

    • Log Tree: Separate B-tree for tracking changes fs/btrfs/tree-log.c111-170
    • Log Replay: Recovery process after crashes fs/btrfs/tree-log.c99-104
    • Optimization: Only logs changed inodes and directories fs/btrfs/tree-log.c228-235

    Sources: fs/btrfs/tree-log.c1-500 fs/btrfs/inode.c123-753

    Modern UFS controllers support MCQ mode for improved parallelism:

    Summary¶

    The Linux storage and I/O subsystems provide a flexible, high-performance stack:

    • io_uring offers modern asynchronous I/O with minimal syscall overhead and zero-copy capabilities
    • Btrfs provides advanced filesystem features with copy-on-write semantics and strong data integrity
    • UFS enables high-performance mobile storage with multi-queue support and platform-specific optimizations

    These subsystems integrate through well-defined interfaces (VFS, block layer, SCSI) while each optimizes for its specific domain. For detailed information about each subsystem's implementation, refer to their dedicated pages: io_uring, Btrfs, and UFS.
