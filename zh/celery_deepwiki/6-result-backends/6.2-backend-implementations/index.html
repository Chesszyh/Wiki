    后端实现概览¶

    Celery 提供了多种后端实现，这些实现扩展了基础后端架构以支持不同的存储系统。每种实现都为结果存储和检索提供了特定的能力和权衡。

    Redis 后端特性¶

    RedisBackend 类 celery/backends/redis.py188-615 提供了全面的结果后端能力：

    特性支持：

    • supports_autoexpire = True —— 基于 TTL 的自动过期 celery/backends/redis.py204
    • supports_native_join = True —— 原生结果合并 (join) celery/backends/redis.py205
    • implements_incr = True —— 通过 incr() 实现原子递增操作 celery/backends/redis.py424-425

    核心操作：

    • get(key) —— 单个键检索 celery/backends/redis.py381-382
    • mget(keys) —— 批量键检索 celery/backends/redis.py384-385
    • set(key, value) —— 带有流水线发布 (pipeline publish) 的存储 celery/backends/redis.py402-415
    • delete(key) —— 键删除 celery/backends/redis.py421-422

    Chord 协调： Redis 后端通过 on_chord_part_return() celery/backends/redis.py469-556 实现了复杂的 chord 协调，使用 Redis 有序集合 (ZADD/ZRANGE) 或列表 (RPUSH/LRANGE) 进行有序的结果收集。

    MongoDB 后端操作¶

    MongoBackend 类 celery/backends/mongodb.py35-334 实现了基于文档的结果存储：

    核心操作：

    • _store_result() —— 使用带有 upsert 的 replace_one() 存储任务结果 celery/backends/mongodb.py181-194
    • _get_task_meta_for() —— 使用 find_one() 检索任务元数据 celery/backends/mongodb.py197-224
    • _save_group() —— 存储组结果 celery/backends/mongodb.py226-234
    • _restore_group() —— 检索组结果 celery/backends/mongodb.py236-247
    • _delete_group() —— 按 ID 删除组 celery/backends/mongodb.py249-251
    • _forget() —— 使用 delete_one() 移除结果 celery/backends/mongodb.py253-263
    • cleanup() —— 删除过期的文档 celery/backends/mongodb.py265-275

    文档结构：每个任务结果都存储为一个 MongoDB 文档，其中 _id 是任务 ID celery/backends/mongodb.py188，并包含标准的元数据字段，如 status, result, date_done 和 traceback。扩展元数据包含 name, args, kwargs, queue, worker 和 retries celery/backends/mongodb.py201-215。

    二进制编解码器处理：该后端通过 BINARY_CODECS = frozenset(['pickle', 'msgpack']) celery/backends/mongodb.py32 处理序列化，并使用 Binary() 包装二进制数据以便在 MongoDB 中存储 celery/backends/mongodb.py172-173。

    连接管理：

    • _get_connection() —— 使用连接选项创建 MongoClient celery/backends/mongodb.py136-163
    • _ensure_mongodb_uri_compliance() —— 验证并规范化 MongoDB URI celery/backends/mongodb.py118-127
    • _prepare_client_options() —— 根据 PyMongo 版本配置客户端选项 celery/backends/mongodb.py129-134

    来源：celery/backends/mongodb.py32 celery/backends/mongodb.py136-163 celery/backends/mongodb.py181-275

    Cassandra 后端特性¶

    CassandraBackend 类 celery/backends/cassandra.py71-257 提供分布式结果存储：

    特性支持：

    • supports_autoexpire = True —— 基于 TTL 的自动过期 celery/backends/cassandra.py86
    • 双连接模式：标准集群或 AstraDB 云捆绑包 celery/backends/cassandra.py104-110
    • 读/写操作的一致性级别配置 celery/backends/cassandra.py117-125

    查询模板：后端将 CQL 查询模板定义为常量 celery/backends/cassandra.py37-64：
