    目的与范围¶

    本文档介绍了 RAGFlow 的多层存储架构，说明了系统如何在四个不同的存储后端之间分离关注点：用于元数据的 MySQL、用于缓存和任务队列的 Redis、用于全文和向量搜索的可插拔文档引擎，以及用于二进制对象存储的 MinIO/S3。

    有关编排跨这些存储层进行数据处理的任务执行系统的信息，请参阅任务执行与队列系统。有关使用这些存储系统的应用服务的详细信息，请参阅核心应用服务。

    数据库配置¶

    MySQL 连接通过环境变量配置，并支持 MySQL 和 PostgreSQL 后端：

    • 主机（Host）: MYSQL_HOST（默认：mysql）
    • 端口（Port）: MYSQL_PORT（默认：3306）
    • 数据库（Database）: MYSQL_DBNAME（默认：rag_flow）
    • 凭据（Credentials）: MYSQL_USER、MYSQL_PASSWORD
    • 连接池: PooledMySQLDatabase 或 PooledPostgresqlDatabase

    api/db/db_models.py700-750 实现了具有自动重连处理功能的数据库连接单例。

    关键数据库模型¶

    RAGFlow 使用 Peewee ORM 为所有主要实体定义了模型。每个模型都扩展了 BaseModel，后者提供自动时间戳管理：

    • User: 用户身份验证和个人资料 api/db/db_models.py800-850
    • Tenant: 多租户隔离，LLM 默认值 api/db/db_models.py850-900
    • Knowledgebase: 知识库配置、解析器设置、嵌入模型 api/db/services/knowledgebase_service.py1-100
    • Document: 文件元数据、解析进度、分块数量 api/db/services/document_service.py46-78
    • Task: 异步解析任务状态 api/db/services/task_service.py1-100
    • Dialog: 聊天助手配置 api/db/services/dialog_service.py50-68
    • TenantLLM: 每个租户的 LLM API 密钥 api/db/db_models.py400-450

    任务队列系统¶

    RAGFlow 使用 Redis Streams 通过消费者组实现可靠的任务排队。该队列架构确保了精确一次（exactly-once）的处理，并能对未确认（unacknowledged）的消息进行自动重试。

    队列配置：

    • 队列名称: 由 settings.get_svr_queue_names() 定义 common/settings.py
    • 消费者组: SVR_CONSUMER_GROUP_NAME 常量 common/constants.py79
    • 消费者名称: task_executor_{CONSUMER_NO} rag/svr/task_executor.py113

    队列操作：

    • REDIS_CONN.queue_product(): 生产新任务到队列 api/db/services/task_service.py200-250
    • REDIS_CONN.queue_consumer(): 从队列中取出并处理 rag/svr/task_executor.py173-234
    • REDIS_CONN.get_unacked_iterator(): 重试未确认的消息 rag/svr/task_executor.py180-183

    来源：rag/svr/task_executor.py173-234 rag/utils/redis_conn.py1-300

    LLM 响应缓存¶

    为了降低 API 成本和延迟，RAGFlow 会为确定性操作（如关键词提取、问题生成和元数据提取）缓存 LLM 响应。

    缓存键结构：

    {llm_name}:{content_hash}:{operation_type}:{params_json}
    被缓存的操作：

    • keyword_extraction: 自动生成的关键词 rag/svr/task_executor.py344-356
    • question_proposal: 自动生成的问题 rag/svr/task_executor.py377-389
    • gen_metadata: 提取的元数据字段 rag/svr/task_executor.py409-424
    • content_tagging: 文档标签 rag/svr/task_executor.py478-500

    缓存函数：

    • get_llm_cache(llm_name, content, cache_type, params) graphrag/utils.py30-50
    • set_llm_cache(llm_name, content, result, cache_type, params) graphrag/utils.py51-70

    来源：rag/svr/task_executor.py344-512 graphrag/utils.py1-100

    分布式锁定¶

    Redis 为跨多个工作线程协调并发操作提供分布式锁。

    锁实现：

    • RedisDistributedLock 类 rag/utils/redis_conn.py200-300
    • 用于数据源同步 rag/svr/sync_data_source.py1-200
    • 防止对同一文件的重复处理

    Redis 配置：
